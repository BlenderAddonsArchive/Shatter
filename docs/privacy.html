Shatter Privacy

Privacy and Security Statement

This is not a legally binding document and it is for informational purposes only.

User IDs

Each install of Shatter is given a random user ID that is generated on your device. This user ID is never submitted to servers, but may be included in your segment's XML file if you have the "Export metadata" option enabled in Shatter preferences.

Connection to external services

Shatter developers do not use online services to store personal information unless you explicitly provide it to us; however, there are some features of Shatter that connect to the internet which may collect personally identifiable information:

  • The automatic updater requests files from GitHub servers in some cases. Please see GitHub's privacy statement.
  • Shatter may download a list of SHA1 user ID hashes to see if your user ID has been blocked from using Shatter.
    • Note: Due to bugs, this is disabled in current versions of Shatter.
  • Some versions of Shatter may have opt-in telemetry options, which will submit crash reports to our logging service.
    • Note: The telemetry endpoints were taken down on 31 July 2023 and will no longer collect information.

Hosting or creating local network services

There are also features which require hosting a webserver, which might be exposed to the public if your network is configured incorrectly:

  • The quick test feature requires hosting a webserver which does not serve anything but files at semi-hardcoded file paths.

Automatic update security

We use code signing to verify that updates come from the original developers of Shatter, even in the event that an account is hacked; however, code signing like this could still fail if an attacker has access to private keys needed to sign Shatter updates. Since Blender addons run an unsandboxed version of python, this means a malicous update could be able to install malicous software, like malware or viruses, if a developer is successfully attacked. You shouldn't enable automatic updates if you are concerned about running bad programs.

Disabling network connectivity and other features

You can disable parts of Shatter that require network connections and similar features by going to the Blender Addon Preferences (Edit → Preferences → Add ons), then finding "Shatter". You can then find options to disable and enable the update notifier, the auto-updater and the quick test server.

How to report security issues

There has currently been no external security audit. If you find an issue or want to preform an audit, please report it to one of the following:
