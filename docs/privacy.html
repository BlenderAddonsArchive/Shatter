Shatter Privacy and Security Statement

Shatter Privacy and Security Statement

Updated 29 October 2023

This is not a legally binding document and it is for informational purposes only.

Telemetry

Shatter does not currently collect telemetry information; however, by continuing to use Shatter, you agree to let us turn on telemetry information by default in the future.

User IDs

Each install of Shatter is given a random user ID that is generated on your device. This user ID is never submitted to servers, but may be included in exported segment files.

Connection to external services

Shatter developers do not use online services to store personal information unless you explicitly provide it to us; however, there are some features of Shatter that connect to the internet which may collect personally identifiable information:

  • The automatic updater requests files from GitHub servers. Please see GitHub's privacy statement.
  • Shatter may download information about blocked users to see if your user ID has been blocked from using Shatter.

Update security

Most downloads, including all of them that contain executeable code, use digital signatures to verify that the recieved data has come from or been authorised by the Shatter developers.

However, we should note that automatic updates are still a risk if both the account is hacked and the private signing key is leaked. If you are concerned about this, you can disable the automatic updater in the addon settings.

Quick test server

There are also features which require hosting a webserver, which might be exposed to the public if your network is configured incorrectly:

  • The quick test feature requires hosting a webserver which does not serve anything but files at semi-hardcoded file paths.

Network settings

You can disable parts of Shatter that require network connections and similar features by going to the Blender Addon Preferences (Edit → Preferences → Add ons), then finding "Shatter". You can then find options to disable and enable the update notifier, the auto-updater and the quick test server as well as other network features.

Reporting security issues

There has currently been no external security audit. If you find an issue or want to preform an audit, please report it to one of the following:

  • Knot126 via Email: Click to see email

If you send an email or DM about non-security related issues, you will most likely be blocked. Please keep discussion public unless it is sensitive.
